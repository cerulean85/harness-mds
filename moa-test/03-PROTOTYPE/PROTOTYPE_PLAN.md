# MOA 핵심 시나리오 프로토타입 계획

## 목적

기존 와이어프레임을 기반으로 핵심 사용자 흐름을 클릭 가능한 MVP로 구현하고, 정상/예외 흐름을 화면 상태로 검증한다.

## 1차 프로토타입 범위

| 우선순위 | 흐름 | 포함 화면 | 검증 포인트 |
| --- | --- | --- | --- |
| P0 | 로그인 및 세션 재인증 | 인증 | 로그인 성공, 실패 누적, 세션 만료 후 재인증 |
| P0 | 근거 기반 질의 응답 | 작업 콘솔 | 질의 입력, DB 자동 선택, 근거/출처/조치 절차 표시 |
| P0 | 자연어 설비 제어 | 작업 콘솔 | 권한 확인, 위험 등급, 재확인, 차단/전송 결과 |
| P0 | 알람 대응 | 알람 센터 | 알람 선택, 처리 가이드, 담당자 통지, 히스토리 |
| P1 | 파일 업로드 및 VectorDB | 자료 관리 | 정책 검증, 차단 사유, 벡터화 상태, 공유 범위 |
| P1 | 통계 분석 및 내보내기 | 분석 | 자연어 분석, 차트/표 결과, PDF/CSV/XLSX 내보내기 |
| P1 | 이력 및 피드백 | 이력 | 피드백 저장, 검색, 소프트 삭제 |
| P2 | 관리자 운영 | 관리자 | 사용자 제지, 모델 버전, 파일 정책, 로그, 백업 상태 |

## 페르소나 검증 세트

| ID | 페르소나 | 주요 관찰 항목 |
| --- | --- | --- |
| P-01 | 신규 사용자 | 화면 진입, CTA 이해, 오류 메시지 이해 |
| P-02 | 숙련 운영자 | 반복 작업 속도, 정보 밀도, 단축 흐름 |
| P-03 | 고령 사용자 | 글자 크기, 버튼 구분, 단계 명확성 |
| P-04 | 보조기술 사용자 | 키보드 이동, 레이블, 상태 전달 |
| P-05 | 저사양/작은 화면 사용자 | 모바일 레이아웃, 핵심 액션 노출, 과밀 여부 |

## 성공 기준

- 핵심 P0 시나리오는 화면 전환 없이 끝까지 클릭 가능해야 한다.
- 각 P0 시나리오는 최소 1개 이상의 예외 상태를 포함해야 한다.
- 시스템 판단에는 사유가 표시되어야 한다.
- 설비 제어 명령은 재확인 또는 차단 상태 없이 바로 전송되면 안 된다.
- 모든 사용자 액션은 화면 하단 실행 로그에 기록되어야 한다.

## 산출물

- `prototype.html`: 클릭 가능한 1차 프로토타입
- `prototype_scenarios.json`: 핵심 시나리오 정의
- `PROTOTYPE_PLAN.md`: 범위, 검증 기준, 페르소나 계획

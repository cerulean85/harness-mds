# MOA 와이어프레임 가이드

## 산출물

- `WIREFRAME_BOARD.html`: 화면 골격, 영역 배치, 이동 흐름 중심의 실제 와이어프레임 보드
- `wireframe.html`: 브라우저에서 바로 열어 검토하는 정적 와이어프레임
- `wireframe_mapping.json`: 화면 ID, 기능 ID, SRS ID, 권한, 주요 컴포넌트 매핑

## 사용 기준

- 구조 검토, 화면 누락 확인, 이해관계자 리뷰에는 `WIREFRAME_BOARD.html`을 사용한다.
- 실제 화면에 가까운 컴포넌트 배치와 샘플 데이터를 보고 싶을 때는 `wireframe.html`을 사용한다.
- 추적성 검토에는 `wireframe_mapping.json`을 사용한다.

## 화면 목록

| 화면 ID | 화면명                  | 주요 기능                              |
| ----- | -------------------- | ---------------------------------- |
| WF-00 | 화면 구조 및 커버리지         | IA, 대표 흐름, 권한별 접근, 기능 커버리지         |
| WF-01 | 인증 화면                | 회원가입, 로그인, 계정 잠금, 세션 재인증           |
| WF-02 | 통합 작업 콘솔             | 질의 응답, 데이터 소스 라우팅, 음성, 설비 제어, 피드백  |
| WF-03 | 알람 센터                | 실시간 알람, 처리 가이드, 이메일/SMS 통지, 히스토리   |
| WF-04 | 자료 업로드 및 VectorDB 관리 | 파일 업로드, 정책 검증, 벡터화, VectorDB 생성    |
| WF-05 | 통계 분석 및 리포팅          | 조건/자연어 분석, 차트/표, PDF/CSV/XLSX 내보내기 |
| WF-06 | 이력 및 피드백             | 질의 이력 조회, 답변 피드백, 소프트 삭제           |
| WF-07 | 관리자 및 운영 화면          | 사용자 제지, 모델 버전, 파일 정책, 로그, 백업/운영    |

## 대표 사용자 흐름

1. 비로그인 사용자가 회원가입 또는 로그인을 수행한다.
2. 인증된 사용자가 통합 작업 콘솔에서 자연어 질의 또는 음성 질의를 입력한다.
3. 시스템이 권한과 의도를 기준으로 데이터 소스를 자동 선택하거나 사용자가 명시한 범위로 제한한다.
4. AI 답변 카드에 요약, 원인/분석, 조치 절차, 근거, 출처, 관련 이미지를 표시한다.
5. 사용자는 답변에 피드백을 남기거나 대화와 분석 결과를 내보낸다.
6. 설비 제어 명령은 권한 확인, 명령 요약, 위험 등급, 재확인 또는 추가 승인 후 전송한다.
7. 관리자는 사용자 제지, 모델 버전, 파일 정책, 로그 검색, 백업 및 운영 상태를 관리한다.

## 검토 포인트

- 일반 사용자와 관리자 화면이 명확히 분리되어 있는지 확인한다.
- 설비 제어 명령의 재확인/승인 단계가 운영 정책과 맞는지 확인한다.
- 알람 처리 가이드, 근거 기반 답변, 분석 결과의 출처 표시 방식이 충분한지 확인한다.
- 파일 정책 차단 사유와 VectorDB 공유 범위가 실제 운영 권한 체계와 맞는지 확인한다.
- `wireframe_mapping.json`에서 모든 `FS-001`~`FS-010`, `SRS-001`~`SRS-039`가 누락 없이 매핑되었는지 확인한다.
